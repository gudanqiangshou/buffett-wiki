# Tina-巴菲特知识库 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Buffett investment knowledge graph website with a structured Markdown wiki, Vue 3 frontend with D3.js visualization, and an AI Buffett chatbot powered by Anthropic API.

**Architecture:** Greenfield project with three layers: (1) a wiki/ folder of interlinked Markdown pages with YAML frontmatter, (2) a Python build pipeline that converts wiki into JSON data files, (3) a Vue 3 SPA consuming those JSON files + an Express backend serving AI chat via SSE. The wiki is the single source of truth; the build pipeline is the bridge; the frontend is read-only.

**Tech Stack:** Vue 3 (Composition API), Vite 6, D3.js 7, markdown-it, Express 4, Anthropic SDK, Python 3 (no deps beyond stdlib + PyYAML)

**Note on source data:** The original source directories (`巴菲特信汇总/`, `巴菲特访谈/`) are not yet present. This plan creates representative sample data (15-20 wiki pages) so the full system can be built and tested end-to-end. When source data becomes available, the ingest pipeline will populate the remaining ~220 pages.

### Task 16: server.js (Express API Server)

**Files:**
- Create: `code/web/server.js`

- [ ] **Step 1:** Express setup:
  - ESM imports, CORS middleware
  - Serve `public/` as static files
  - Port 3001

- [ ] **Step 2:** Startup data preload:
  - Load `public/data/wiki-index.json` into memory
  - Load `public/data/graph.json` into memory
  - Build adjacency map (neighbors) from edges
  - Build title index (indexByTitle) for fast lookup

- [ ] **Step 3:** `POST /api/verify-password`:
  - Compare request body password against `process.env.ACCESS_PASSWORD`
  - Return `{valid: true/false}`

- [ ] **Step 4:** Retrieval algorithm — Phase 1 (multi-granularity matching):
  - Generate 2-4 char sliding window ngrams from question (Chinese-friendly)
  - Keep space-separated complete words (English-friendly)
  - Scoring:
    - Title fully contained in question → +50
    - Question contained in title → +40
    - Ngram hits title → +8
    - Ngram hits summary → +3
    - Links keyword hits question → +6
  - Take TOP 4

- [ ] **Step 5:** Retrieval algorithm — Phase 2 (graph expansion):
  - For each TOP 4 hit, find neighbors in graph adjacency map
  - Neighbor must have relevance score > 0 to question
  - Add up to 2 related pages
  - Final: 4-6 highly relevant pages

- [ ] **Step 6:** Context builder:
  - Read matched pages from `public/data/pages/<category>/<slug>.md`
  - Take first 3000 chars per page
  - Assemble into context string

- [ ] **Step 7:** `POST /api/chat` — SSE endpoint:
  - Validate password header/session
  - Run retrieval on latest user message
  - Call Anthropic SDK `.messages.stream()` with:
    - System prompt: Buffett-style persona, use provided context
    - Retrieved page context
    - Conversation history
  - Stream response as SSE: `data: {"text":"chunk"}\n\n`
  - End with `data: {"done":true,"sources":["page1","page2"]}\n\n`
  - Use `ANTHROPIC_BASE_URL` and `ANTHROPIC_MODEL` env vars

- [ ] **Step 8:** Commit: `feat: implement Express server with retrieval and SSE chat`
